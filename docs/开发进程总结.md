# 点云预处理 Baseline 开发进程总结

> 记录从"完全照搬东北大学 baseline"到当前 4/5mm 调优版本的迭代过程。
> 每个版本说明:**解决什么问题 → 怎么解决 → 结果数据**。

## 数据与目标

- 输入:工业 3D 相机采集的两幅点云(PCL 二进制 PLY,单位 mm)
  - `origin-only-environment.ply`(空环境)— 9,542,897 点
  - `raw_subass_workpiece.ply`(含工件)— 9,450,388 点
- 目标:去除背景与噪声,提取出干净的工件点云 `workpiece_clean.ply`
- 核心思想:`含工件场景点云 - 空环境点云 = 工件点云`

---

## 版本 0:照搬东北大学 baseline(初版骨架)

### 解决的问题
搭出可运行的传统几何 pipeline 骨架,验证流程可行性。

### 做法
按论文流程实现,模块化 C++(一个模块一个 cpp),纯标准库无第三方依赖:

```
ROI 裁剪 → 体素下采样 → 统计离群点去除 → 差异点云分割
```

- `roi_crop`:轴对齐包围盒裁剪
- `voxel_downsample`:体素下采样,取"距体素重心最近的原始点"(保留真实采样点,利于边缘细节)
- `statistical_filter`:统计离群点去除(kNN 平均距离 + 均值/标准差阈值)
- `diff_segmentation`:对 scene 每个点在 env 中找最近邻,距离 > 阈值判为工件
- 先用脚本生成的小规模合成点云跑通,ROI/参数为占位值

### 结果
流程跑通,但仅在合成数据上验证,参数未针对真实数据。

## 版本 1:适配真实工业数据

### 解决的问题
换上真实数据后暴露三个问题:
1. 真实文件是 **binary_little_endian PLY**,原读取只支持 ASCII,读不进来。
2. 数据是**毫米量级、约 950 万点**,占位 ROID/参数完全不匹配。
3. 体素下采样把工件削掉了一大半。

### 做法
- **PLY 读写**:扩展 `ply_io`,支持 `binary_little_endian`;大点云(>50万)保存时自动用二进制,避免写 ASCII 过慢。
- **ROI 重设**:分析 env 的工作台层(z∈[-120,-20] 点最密),取 5–95% 分位 + 边距,得到
  `x[-176.4, 1164.9] y[-111.2, 960.7] z[-150, 100]`(mm)。env→3,079,176,scene→3,062,029。
- **体素尺寸**:实测 2mm 体素工件点只剩约 **49%**(稀疏薄壁被合并);改为 **1mm**,工件保留约 **84%**。
- **统计滤波加速**:初版是暴力 O(n²) kNN,在百万级点上**跑了约 5.1 小时**;改为 **KD-tree** 加速。
- **参数初调**:`nb_neighbors 30→15`、`std_ratio 1.5→2.0→4.0`(放宽,避免误删稀疏工件边缘)。

### 结果
能在真实数据上跑完整流程并提取出工件,但**整体耗时约 212 秒**,瓶颈在统计滤波。

## 版本 2:流程重构 + 性能优化

### 解决的问题
- 全流程 ~212s,其中统计滤波在 env+scene 各 230 万点上跑,**单独占约 196s**。
- env 做统计滤波反而有害:删掉 env 点会让对应位置的 scene 噪声在差分时被误判为工件。

### 做法(四项叠加)
1. **重排流程**:把差分提到统计滤波**之前**。新顺序:
   `ROI → 体素 → 差分 → 统计滤波 → (聚类)`。
   差分先把点数从 ~237 万砍到 ~32 万,统计滤波只作用于工件点。
2. **env 跳过统计滤波**:env 只做到体素,直接建 KD-tree 参与差分,既省时又避免误判。
3. **kNN 有界最大堆**:用大小为 k 的最大堆维护最近 k 个平方距离,堆顶剪枝,复杂度从
   O(访问点数 × k) 降到 O(访问点数 × log k)。
4. **统计滤波多线程**:每点查询互相独立、KD-tree 只读,用 `std::thread` 按区间并行跑满多核。

### 结果
全流程从 **~212s 降到 ~14s(约 15×)**;统计滤波从 ~196s 降到 ~0.2s。

| 步骤 | 耗时 |
| --- | --- |
| ROI | ~28 ms |
| 体素下采样 | ~1.5 s |
| 差分分割 | ~9–10 s |
| 统计滤波 | ~0.2 s |
| 聚类去游离点 | ~0.8 s |

## 版本 3:聚类去游离点

### 解决的问题
差分结果里,离工件较远处仍有几条**独立的细长游离条带**(背景噪声/扫描伪影)。
统计滤波删不掉它们(它们局部密度够、不算离群)。

### 做法
新增 `cluster_filter` 模块:欧氏聚类(区域生长),以 3mm 邻接半径把点连成簇,
`keep_largest=1` **只保留最大的那个簇**。

诊断数据(3mm 半径):最大簇 315,274 点(98.8%),其余是 1965/665/327/326/184/126/121… 的小簇,
正好是那些游离条带。

### 结果
`324,683 → 315,275`,删掉约 9,400 个游离条带点,工件主体完整保留。

### 局限(引出版本 4)
工件**附近、紧贴工件**的密集点(边界晕圈)删不掉:
连通分析显示它们在 3D 上与主体连通(3mm 半径下整团是一个连通块),
缩小半径到 2mm 能切断,但工件本体自身也碎成多块(最大块只剩 37.7%)。
所以"按连通性"的聚类无法分离连体晕圈。

## 版本 4:差分阈值调优,去边界晕圈(当前版本)

### 解决的问题
工件附近的连体"白条/晕圈"——视觉上像分开,实际 3D 连通,聚类删不掉。

### 定位(用数据排除误判)
- env 与 scene **对齐良好**:scene 到 env 距离中位数仅 **0.69mm**,86% 的点 <3mm。
  → 不是整体错位,**ICP 配准用不上**。
- 这层晕圈是**差分在"工件/台面边界"处产生的**:边界点离 env 有 3–6mm,被阈值误判为工件。
  workpiece_diff 中约 **39%** 的点落在 3–6mm 这层壳里,且 [3,4)/[4,5)/[5,6) 三段点数几乎相等。

### 做法
既然聚类事后删不掉,就**从源头掐掉**:调大 `diff_threshold`,让"离 env 仅几毫米"的边界点
在差分阶段就被判为背景。工件主体是抬高结构(离 env >10mm),不受影响。

### 各阈值最终结果对比

| diff_threshold | 差分后 | 最终 clean | z 范围 | 备注 |
| --- | --- | --- | --- | --- |
| 3mm | 327,281 | 315,275 | [-77, 65] | 晕圈明显 |
| **4mm** | 284,489 | **277,279** | — | **保留(去晕圈/保底边较平衡)** |
| **5mm** | 239,691 | **230,468** | [-75, 65] | **保留(更干净)** |
| 6mm | 199,121 | 185,982 | [-74, 65] | 仅作对照,过度削减 |

z 顶部始终是 65(工件结构未被动),提阈值主要削掉贴台面的边界晕圈。**最终保留 4mm 与 5mm 两个候选版本。**

### 局限
调大阈值是"一刀切",会连带删掉真实工件中**离背景 <阈值**的点(主要是贴台面的底边)。
若要"既删晕圈又不伤底边",需用基于几何形状的方法(见下)。

## 当前完整流程

```
env / scene (各 ~950 万点)
        │
①  ROI 裁剪          env/scene 同一包围盒                 ~28ms
        ▼
②  体素下采样 1mm     取距重心最近的原始点                 ~1.5s
        ▼
③  差分分割          scene 每点在 env 中找最近邻,           ~9s
        │            距离 > diff_threshold(4/5mm) 判为工件
        ▼
④  统计滤波          仅作用于工件点,多线程 kNN             ~0.2s
        ▼
⑤  聚类去游离点       欧氏聚类,只保留最大簇                ~0.8s
        ▼
   workpiece_clean.ply(最终结果)
```

中间产物均会落盘:`*_roi.ply`、`*_voxel.ply`、`workpiece_diff.ply`、`workpiece_sor.ply`、
`workpiece_clean.ply`,便于逐步排查。

## 当前默认参数(均可命令行覆盖)

| 参数 | 命令行 | 默认值 | 说明 |
| --- | --- | --- | --- |
| ROI 包围盒 | `--roi` | 见上 | 基于 env 工作台层估计 |
| 体素边长 | `--voxel` | 1.0 mm | 减小以保留稀疏工件 |
| 差分阈值 | `--diff-threshold` | 3.0 mm | **实测建议 4–5mm** |
| 统计滤波邻居数 | `--nb-neighbors` | 15 | |
| 统计滤波标准差倍数 | `--std-ratio` | 4.0 | 较宽松 |
| 聚类邻接半径 | `--cluster-radius` | 3.0 mm | |
| 保留最大簇数 | `--keep-largest` | 1 | 去远处独立条带 |
| 最小簇点数 | `--min-cluster-size` | 100 | 仅 keep-largest=0 时生效 |

示例:
```bash
./baseline --step all --diff-threshold 5 --output output/diff_t5
```

## 遗留问题与下一步

1. **连体边界晕圈的"治本"方案**:当前靠调大差分阈值"一刀切",会牺牲贴台面底边。
   更精准的做法是 **RANSAC 平面提取**——工件是平整梁/板面,可被平面模型拟合,
   而晕圈散乱不成面会被丢弃;即使与工件连通,也能按"是否在平面上"区分。
   这也是东北大学论文工件提取后的标准下一步,衔接焊缝特征提取。
2. **底边权衡**:在 4mm 与 5mm 之间最终定一个值(看是否需要保留贴台面底边)。
3. **差分阶段约占全流程 ~9s**,后续若需提速可优化(如对 env 树做近似最近邻)。
