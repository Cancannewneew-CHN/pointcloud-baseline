# GUI 集成接口说明

本文档面向把点云预处理集成进 GUI 的开发者,说明:**两条处理分支、各自的参数接口、自动选择逻辑,以及界面交互建议**。

---

## 一、总体结构:两条分支

GUI 顶层让用户先选择处理分支(二选一):

```
                    ┌─────────────────────────────────────────────┐
   用户选择分支 ──> │  分支 A: 有环境点云 (差分方案)   → baseline   │
                    │  分支 B: 无环境点云 (去台面方案) → tabletop   │
                    └─────────────────────────────────────────────┘
```

| | 分支 A:差分方案 | 分支 B:去台面方案 |
| --- | --- | --- |
| 可执行程序 | `baseline` | `tabletop` |
| 适用场景 | **有**空环境点云(可拍无工件的背景) | **没有**环境点云,但工件放在已知工作台上 |
| 背景剔除原理 | 场景点云 − 空环境点云(最近邻差分) | 去除工作台平面 |
| 必需输入 | 环境点云 + 含工件点云(需同坐标系) | 单个含工件点云 |
| 输入格式 | PLY | PCD / PLY |

两条分支**后半段流程完全相同**(体素下采样 → 统计滤波 → 聚类),只是背景剔除那一步不同。

---

## 二、分支 B(去台面)内部的方法自动选择

分支 B 里有两种去台面方法,**根据用户是否填写 z 切割高度自动选择**:

```
   用户是否填写了 "z 切割高度(z-cut)" ?
        │
        ├── 填了  ──> zcut    : 直接保留 z > 该高度的点 (简单, 快)
        │
        └── 没填  ──> ransac  : 自动拟合工作台平面并去除 (无需任何高度信息)
```

- **zcut**:用户已知台面高度、想要简单可控的平切时使用。GUI 上对应"工作台高度 / 切割高度"输入框,填了就走这条。
- **ransac**:用户不想/不知道填高度时使用。算法自动在点云里找到最大的近水平平面(即台面)并去除,**不需要高度信息**。可选地填一个"台面高度提示"来加速或防止误拟合,但不是必需。

> 经验:zcut 保留贴近台面的点更多(底部"裙边"略大);ransac 更贴合真实台面、底部更干净。两者工件主体一致。

## 三、参数接口清单

### 分支 A(`baseline`,差分)参数

| 接口 | 命令行 | 类型 | 默认 | 暴露档位 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 环境点云 | `--env` | 路径 | — | A | 空场景 PLY |
| 含工件点云 | `--scene` | 路径 | — | A | 含工件 PLY |
| 输出目录 | `--output` | 路径 | output/baseline | A | |
| 差分阈值 | `--diff-threshold` | float(mm) | 3.0(建议4–5) | A | 背景剔除阈值 |
| ROI 包围盒 | `--roi` 6×float | 6×float | 工作台层 | B | x/y/z 裁剪 |
| 运行到哪步 | `--step` | 枚举 | all | C | 单步调试 |

### 分支 B(`tabletop`,去台面)参数

| 接口 | 命令行 | 类型 | 默认 | 暴露档位 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 输入点云 | `--input` | 路径(.pcd/.ply) | — | A | 含工件点云 |
| 输出目录 | `--output` | 路径 | output/tabletop | A | |
| **z 切割高度** | `--z-cut` | float(mm) | 不填 | A | **填了→zcut;不填→ransac** |
| 台面高度提示 | `--table-z` | float(mm) | 不填 | B | 仅 ransac 用,可选,加速/防误拟合 |
| 平面厚度 | `--plane-thickness` | float(mm) | 5 | B | 仅 ransac 用 |
| ROI 包围盒 | `--roi` 6×float | 6×float | 不启用 | B | 可选 x/y/z 裁剪 |

### 两分支公共参数(后半段流程,界面可共用同一组控件)

| 接口 | 命令行 | 类型 | 默认 | 暴露档位 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 体素边长 | `--voxel` | float(mm) | 1.0 | A | 下采样粒度 |
| 保留最大簇数 | `--keep-largest` | int | 1 | A | 工件连通块数 |
| 统计滤波倍数 | `--std-ratio` | float | A:4.0 / B:3.0 | B | 越小越激进 |
| 统计滤波邻居数 | `--nb-neighbors` | int | 15 | B | kNN 数 |
| 聚类半径 | `--cluster-radius` | float(mm) | 3.0 | B | 连通邻接半径 |
| 最小簇点数 | `--min-cluster-size` | int | 100 | B | 仅 keep-largest=0 时生效 |

**暴露档位**:A=主面板(必须暴露);B=高级设置(可折叠);C=可不暴露/内部常量。

## 四、GUI 界面布局建议

### 主面板(A 档)

1. **分支选择**(单选):有环境点云 / 无环境点云。
2. **文件输入**:
   - 分支 A:环境点云 + 含工件点云两个文件框;
   - 分支 B:单个点云文件框。
3. **背景剔除参数**:
   - 分支 A:`差分阈值`(滑块 1–10mm);
   - 分支 B:`z 切割高度`(数字框,**留空表示用 RANSAC 自动**,旁边给个提示)。
4. **公共参数**:`体素边长`(滑块 0.5–5mm)、`保留最大簇数`(数字框 0–10)。
5. **输出目录**。

### 高级设置(B 档,默认折叠)

- 分支 B 专属:`台面高度提示 table-z`、`平面厚度 plane-thickness`。
- 公共:`std-ratio`、`nb-neighbors`、`cluster-radius`、`min-cluster-size`、`ROI`。

### 联动 / 禁用逻辑

1. **分支切换**:选分支 A 时显示 env/scene + diff-threshold,隐藏去台面相关;选分支 B 时反之。公共参数始终显示。
2. **z-cut 留空检测**:分支 B 的 `z 切割高度` 为空 → 走 ransac,启用 `table-z`/`plane-thickness`;填了值 → 走 zcut,灰掉 `table-z`/`plane-thickness`。
3. **聚类模式互斥**:`keep-largest ≥ 1` → 灰掉 `min-cluster-size`;`= 0` → 启用。

## 五、调用与回传(后端对接)

### 命令行调用示例

```bash
# 分支 A: 差分
./baseline --env env.ply --scene scene.ply --diff-threshold 5 --output out/A

# 分支 B: 不填 z → RANSAC 自动去台面
./tabletop --input cloud.pcd --output out/B_ransac

# 分支 B: 填 z=-42 → zcut
./tabletop --input cloud.pcd --z-cut -42 --output out/B_zcut
```

### 回传给 GUI 的信息

两个程序都会:

1. **逐步打印点数**(stdout),格式 `[步骤]: 输入 -> 输出 points, 耗时 ms`,GUI 可解析用于显示统计;
2. **落盘各步骤中间点云**,GUI 可读取做分步预览:
   - 分支 A:`env_roi` / `scene_roi` / `*_voxel` / `workpiece_diff` / `workpiece_sor` / `workpiece_clean`;
   - 分支 B:`02_no_table` / `03_voxel` / `04_sor` / `05_workpiece_clean`。

最终结果统一是 `*workpiece_clean.ply`(或 `05_workpiece_clean.ply`)。

### 接口抽象建议

GUI 不要直接拼命令行字符串,建议后端把参数封装成配置结构体(分支 A 对应 `PipelineConfig`,分支 B 对应 `PlaneParams` + 公共的 voxel/SOR/聚类参数),填好后调用处理函数,回传上述统计与中间点云路径。这样界面与命令行解耦,后续加参数也更容易维护。
