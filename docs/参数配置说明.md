# 参数配置说明

本文档解释 baseline 点云预处理流程中各参数的含义、默认值、作用与调参建议。
所有参数均可通过命令行覆盖,默认值定义在 `baseline/main.cpp` 的 `default_config()`。

流程顺序:`ROI 裁剪 → 体素下采样 → 差分分割 → 统计滤波 → 聚类去游离点`

---

## ① ROI 包围盒 `--roi`

- **含义**:轴对齐包围盒(Axis-Aligned Bounding Box),只保留盒内的点,裁掉视野里无关的地面/墙体/远景。
- **默认值**:`x[-176.4, 1164.9] y[-111.2, 960.7] z[-150, 100]`(mm),基于 `origin-only-environment.ply` 的工作台层分析得到(取工作台所在层 5–95% 分位再留边距)。
- **作用**:第一步就把约 950 万点裁到约 300 万,大幅减小后续计算量;同时排除背景干扰。
- **调参建议**:
  - ROI 必须与 **env 同坐标系**。若换了新数据且坐标系不同,需重新统计其工作台层范围。
  - 太小会切掉工件边缘;太大则保留多余背景、拖慢差分。
  - 六个数依次是 `xmin xmax ymin ymax zmin zmax`。

## ② 体素边长 `--voxel`

- **含义**:把空间切成边长 `voxel` 的立方体小格,每格只保留一个代表点(取最接近该格重心的原始点),实现下采样去冗余。
- **默认值**:`1.0 mm`。
- **作用**:降低点密度、统一采样间隔,加速后续步骤。
- **调参建议**:
  - 实测 `2mm` 会把稀疏薄壁工件削掉约一半,故降到 `1mm`(工件保留约 84%)。
  - 越小越保细节但点更多、更慢;越大越快但易丢薄结构。
  - 建议保持 `diff_threshold ≈ 3 × voxel`。

## ③ 差分阈值 `--diff-threshold`

- **含义**:差分分割(背景剔除)的核心阈值。对场景每个点,在环境点云里找最近邻,**距离 > 阈值**则判为"工件",否则视为背景剔除。
- **默认值**:`3.0 mm`。**实测推荐 4–5mm。**
- **作用**:把"场景中环境里没有的部分"提取出来,即工件。
- **调参建议**:
  - 太小:工件/台面边界处的点离环境只有几毫米,会被误判为工件,形成贴着工件的"边界晕圈"。
  - 太大:会连带删掉工件中靠近背景的真实点(主要是贴台面的底边)。
  - 经验:`3mm` 晕圈明显;`4mm` 较平衡;`5mm` 更干净但削底边;`6mm` 过度削减(仅作对照)。
  - **前提**:env 与 scene 必须配准在同一坐标系且对齐良好(中位残差 < 1mm 级别),否则差分会大面积误判。

## ④ 统计滤波 `--nb-neighbors` / `--std-ratio`

- **含义**:统计离群点去除(SOR)。对每个点计算其到最近 `nb-neighbors` 个邻居的平均距离;统计所有点该距离的均值 μ 与标准差 σ,凡平均距离 `> μ + std_ratio × σ` 的点判为离群点删除。
- **默认值**:`nb-neighbors = 15`,`std-ratio = 4.0`。
- **作用**:删除密度异常稀疏的孤立噪点。
- **调参建议**:
  - `std-ratio` 越小越激进(删得多)。初版 `1.5` 会误删稀疏工件边缘,放宽到 `4.0` 后较安全。
  - `nb-neighbors` 越大估计越稳但越慢;`15` 在该数据上够用。
  - 本步已用 KD-tree + 多线程加速,数十万点仅需百毫秒级。

## ⑤ 聚类去游离点 `--cluster-radius` / `--keep-largest` / `--min-cluster-size`

- **含义**:欧氏聚类。以 `cluster-radius` 为邻接半径做区域生长,把空间上连通的点归为同一簇,再按规则保留/丢弃簇。
- **默认值**:`cluster-radius = 3.0 mm`,`keep-largest = 1`,`min-cluster-size = 100`。
- **两种保留模式**:
  - `keep-largest >= 1`(默认):只保留点数最大的 N 个簇,适合工件是单一/少数连通体的情形,可去掉远处独立的游离条带/噪声簇。**默认 N=1,即只留最大的那一个连通块。**
  - `keep-largest = 0`:改用 `min-cluster-size` 规则,保留所有点数 ≥ 该值的簇。
- **调参建议**:
  - `cluster-radius` 太小会把工件本体打碎成多块;太大则把附近噪声也连进主簇。`3mm` 与体素/差分阈值匹配。
  - 工件若本身分成几块,把 `keep-largest` 调成对应数量(如 `2`)。
  - 注意:聚类**无法去除与工件 3D 连通的"边界晕圈"**——那类噪声要靠调大 `diff-threshold` 从源头处理。

## 运行控制参数

| 选项 | 说明 |
| --- | --- |
| `--step` | 运行到哪一步:`roi` / `voxel` / `diff` / `sor` / `cluster` / `all`(默认 `all`)。便于单步调试。 |
| `--env` | 空环境点云路径(默认 `data/origin-only-environment.ply`)。 |
| `--scene` | 含工件场景点云路径(默认 `data/raw_subass_workpiece.ply`)。 |
| `--output` | 输出目录,自动落盘各步骤中间产物。 |

---

## 当前推荐配置(示例)

```bash
# 默认完整流程
./baseline/baseline --step all --output output/baseline

# diff 用 5mm 更干净 (去边界晕圈)
./baseline/baseline --step all \
  --diff-threshold 5 \
  --output output/diff_t5

# 只看 ROI 裁剪结果, 单步调试
./baseline/baseline --step roi --output output/baseline
```

> 提示:差分要求 env 与 scene 处于**同一坐标系且对齐良好**(中位最近邻残差 < 1mm 级别),
> 否则任何阈值都无法得到干净工件。若换用新采集的数据,需确认其坐标系与 env 一致,
> 并据此重新核对 ROI 包围盒范围。
